# Okafor Hospital Management System - Development Prompt

## Project Overview
**Okafor** is a comprehensive hospital management system built with ASP.NET Core MVC (.NET 10) designed to streamline patient care, appointments, teleconsultations, billing, and admin operations. It serves hospitals across Africa with localized payment and notification providers.

**Repository**: Okafor-.NET  
**Tech Stack**: ASP.NET Core MVC, Entity Framework Core 10.0.7, SQL Server (LocalDB dev), xUnit 2.9.2, GitHub Actions CI/CD  
**Status**: Infrastructure complete (tests ✓, CI/CD ✓, logging ✓); feature development in progress

## Development Standards & Patterns

### Testing Requirements
**All new features must include:**
- Unit tests (models, service logic) with >80% coverage
- Integration tests (controllers, end-to-end flows)
- Use xUnit for test framework
- Use `WebApplicationFactory` for integration tests
- Mock external services (payments, notifications, video platforms)
- Test data in `tests/Okafor.NET.Tests/` folder (NOT main project)

**Example Test Structure:**
```csharp
public class FeatureServiceTests
{
    [Fact]
    public async Task ExecuteFeature_WithValidInput_ReturnsExpected()
    {
        // Arrange
        var service = new FeatureService(_mockDependency);
        
        // Act
        var result = await service.ExecuteFeatureAsync(input);
        
        // Assert
        Assert.NotNull(result);
    }
}
```

### Code Quality Checklist
- [ ] No compiler warnings (nullable, unused imports, etc.)
- [ ] CI/CD pipeline passes (`dotnet build`, `dotnet test`, `dotnet list package --vulnerable`)
- [ ] New tests added and passing
- [ ] Logging added for business logic (use `ILogger<T>`)
- [ ] Database migrations created (EF Core)
- [ ] README updated with feature documentation
- [ ] GitHub Actions CI reflects changes

### Logging Pattern
```csharp
public class FeatureService
{
    private readonly ILogger<FeatureService> _logger;
    
    public async Task ExecuteAsync(string input)
    {
        _logger.LogInformation("Starting feature execution with input: {Input}", input);
        try
        {
            // ... logic ...
            _logger.LogInformation("Feature executed successfully");
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "Feature execution failed");
            throw;
        }
    }
}
```

### Dependency Injection Pattern
All services registered in `Program.cs`:
```csharp
builder.Services.AddScoped<IAppointmentService, AppointmentService>();
builder.Services.AddScoped<INotificationService, LeanNotificationService>(); // Or production provider
```

### Database Migration Pattern
```bash
# After adding/modifying models:
dotnet ef migrations add MigrationName
dotnet ef database update
```

### Environment-Specific Logic
```csharp
if (app.Environment.IsProduction())
{
    // Production: Real payment gateway, production notifications
}
else if (app.Environment.IsEnvironment("Testing"))
{
    // Testing: In-memory DB, mock services
}
else
{
    // Development: LocalDB, sandbox providers
}
```
